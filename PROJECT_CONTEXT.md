# PROJECT_CONTEXT

## 제품 방향
- 현재 제품은 OCR 기반이 아니라 **게시글 제목 양식 기반** 대시보드입니다.
- 공식 입력 경로는 게시글 제목 `거리 / 시간` 형식입니다.
- 기본 해석 규칙은 다음과 같습니다.
  - 앞 숫자 = 거리
  - 뒤 숫자 = 시간
  - 거리는 항상 미터(`m`)
- 공식 표준 제목 예시는 `1500 / 42:30` 입니다.

## 허용하는 제목 예시
- `1500 / 42:30`
- `1500m / 42:30`
- `1500 / 42분 30초`
- `1500 / 1시간 05분`
- `1500m / 55분`

## 허용하지 않는 것
- `1.5km / 42:30`
- `42:30 / 1500`
- `1500 / 페이스 2:05`
- `1500 / 09:10~10:05`
- 숫자가 많이 섞여 모호한 제목

## 데이터 정책
- 집계 시작일: `2026-03-01`
- `2026-03-15`까지는 `2026-03-01` 이후 글 수정분을 다시 수집
- `2026-03-16`부터는 최근 3일 글만 다시 수집
- 공식 스케줄: 매일 `10:00`, `22:00`
- 공식 갱신 명령: `python -m swimdash refresh`

## 대시보드 방향
- 메인 화면은 운동 정보, 랭킹, 배지/칭호를 먼저 보여줍니다.
- 비주얼 톤은 stickerbook, 정보 구조는 game dashboard 입니다.
- Top 3는 크게, 4~20위는 닉네임이 계속 보이게 설계합니다.
- 개인 페이지는 “내 수영 앨범” 느낌으로 갑니다.

## 배지/칭호
- 배지/칭호는 `data/admin/*.json` 설정 파일로 관리합니다.
- 갤 전체 배지는 50km 단위 + 주요 milestone 특별칭호 구조입니다.
- 개인 대표 칭호는 자동 계산 + 관리자 override 가능 구조입니다.

## 관리자 페이지
- `/admin.html` 는 로그인 후 접근합니다.
- 인증은 공유 비밀번호 1개 + 세션 쿠키 방식입니다.
- 저장 대상은 `data/admin/*.json`
- 공개 페이지가 읽는 `docs/data/*.json` 은 rebuild 후 갱신됩니다.
- 이번 단계의 관리자 페이지는 실제 저장까지 지원합니다.

## 배포
- 목표 배포 환경은 Synology NAS + Docker/Container Manager 입니다.
- 스케줄은 DSM Task Scheduler에서 `10:00`, `22:00` 두 번 실행합니다.
- 외부 공개 링크는 로컬 테스트용 Quick Tunnel 일 수 있으며 재시작 시 바뀔 수 있습니다.
